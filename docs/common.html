rotzyx

common.rotzyx(psi: float, theta: float, phi: float) -> numpy.ndarray

parameter type description
phi float rotation angle around the X axis [rad]
theta float rotation angle around the Y axis [rad]
psi float rotation angle around the Z axis [rad]

returns 3D-rotation matrix based on ZYX Euler angles as a numpy.ndarray. The angular parameters are in radian.

$$ \mathrm{rotzyx}(\psi,\theta,\phi) = \left( \begin{array}{ccc} 1.0 & 0.0 & 0.0 \\ 0.0 & \cos \phi & -\sin \phi \\ 0.0 & \sin \phi & \cos \phi \end{array} \right) \left( \begin{array}{ccc} \cos \theta & 0.0 & \sin \theta \\ 0.0 & 1.0 & 0.0 \\ -\sin \theta & 0.0 & \cos \theta \end{array} \right) \left( \begin{array}{ccc} \cos \psi & -\sin \psi & 0.0 \\ \sin \psi & \cos \psi & 0.0 \\ 0.0 & 0.0 & 1.0 \end{array} \right) $$

In each FD element (except for polygons), the local origin and base vectors are defined inside .Basetrans (type: OpenTDv62.Matrix3d). OpenTDv62.Matrix3d offers several methods to set the origin and rotation. However, it is also possible to diretly set the Matrix3d.entry property. In Matrix3d.entry, 0–2nd colums are the base vectors and 3rd column is the origin. The calculated matrix with this function can be used to specify the base vectors as show in the example below: 

new_base = common.rotzyx(psi, theta, phi)
rectangle = td.CreateRectangle()
rectangle.BaseTrans.entry[0][0] = new_base[0,0]
rectangle.BaseTrans.entry[1][0] = new_base[1,0]
rectangle.BaseTrans.entry[2][0] = new_base[2,0]
rectangle.BaseTrans.entry[0][1] = new_base[0,1]
rectangle.BaseTrans.entry[1][1] = new_base[1,1]
rectangle.BaseTrans.entry[2][1] = new_base[2,1]
rectangle.BaseTrans.entry[0][2] = new_base[0,2]
rectangle.BaseTrans.entry[1][2] = new_base[1,2]
rectangle.BaseTrans.entry[2][2] = new_base[2,2]

get_number_of_nodes

common.get_number_of_nodes(td: OpenTDv62.ThermalDesktop, submodel_name: str='') -> int

parameter type description
td OpenTDv62.ThermalDesktop Thermal Desktop instance
submodel_name = '' str name of the submodel

By default, the function returns the number of nodes of the entire model. If 'submodel_name' is given, it returns the number of nodes of the specified submodel.

screenshot

common.screenshot(td: OpenTDv62.ThermalDesktop, name: str, view: str='', style: str='XRAY', imageformat: str='png', width: int=0, height: int=0)

parameter type description
td OpenTDv62.ThermalDesktop Thermal Desktop instance
name str name for the image file
view = '' str available perspectives: ‘SW’, ‘SE’, ‘NE’, ‘NW’, ‘TOP’, ‘BOTTOM’, ‘FRONT’, ‘BACK’, ‘LEFT’, ‘RIGHT’
style = '' str By default, the screen shot will be taken without changing the perspective.
available model styles: ‘WIRE_2D’, ‘WIRE’, ‘CONCEPTUAL’, ‘HIDDEN’, ‘REALISTIC’, ‘SHADED’, ‘SHADED_W_EDGES’, ‘SHADES_OF_GREY’, ‘SKETCHY’, ‘THERMAL’, ‘THERMAL_PP’, ‘XRAY’.
imageformat str By default, png format image will be created.
available image format: ‘bmp’, ‘emf’, ‘exif’, ‘gif’, ‘icon’, ‘jpeg’, ‘png’, ‘tiff’, ‘wmf’.
width = 0 int width pixel for trimming
height = 0 int height pixel for trimming

If ‘width’ and/or ‘height’ is smaller than the original screenshot, the image is trimmed (not scaled) to the specified size. The reference point for the trimming is the center of the original screenshot.
